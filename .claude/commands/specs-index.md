Regenerate `specs/INDEX.md` grouping specs by status.

## Steps

1. List all active specs: `specs/NNN-*/spec.md` (ignore `archive/`, `README.md`, `INDEX.md`).
2. List all archived specs: `specs/archive/*/NNN-*/spec.md`.
3. For each spec, read the header and extract:
   - `# Feature: <title>` (line 1)
   - `**Status:** <status>` (first lines)
   - `**Created:** YYYY-MM-DD`
4. Group by status and write `specs/INDEX.md` following the structure below.
5. Update `**Last updated:** <today>` at the top.

## Required INDEX.md structure

```markdown
# Specs Index

Overview of project specs. Regenerated by `/specs-index`.

**Last updated:** YYYY-MM-DD

## In progress

| ID | Feature | Started |
|----|---------|---------|
| NNN | [Title](NNN-slug/spec.md) | YYYY-MM-DD |

## Approved (next to implement)

| ID | Feature | Approved |
|----|---------|---------|
| ... | ... | ... |

## Draft (being written)

| ID | Feature | Created |
|----|---------|--------|
| ... | ... | ... |

## Done (last 30 days)

| ID | Feature | Completed |
|----|---------|----------|
| ... | ... | ... |

## Archived

| ID | Feature | Quarter |
|----|---------|--------|
| 001 | [Text filter](archive/2026-Q1/001-text-filter/spec.md) | 2026-Q1 |

---

## Status convention
| Status | Meaning |
|--------|---------|
| `draft` | Being written, still has `[NEEDS CLARIFICATION]` |
| `approved` | Ready for `/plan` |
| `in-progress` | `/implement` running |
| `done` | Feature in production |
| `abandoned` | Cancelled or superseded |
```

## Rules

- Empty sections must show `_(none)_` in italics.
- Done > 30 days ago goes to "Archived" if already physically in `archive/`. If still in `specs/NNN-*/` but with an old date, suggest running `/specs-archive`.
- Don't touch the spec contents — only read.
- After writing the INDEX, report: total active / archived, and which section was empty.
